PoX-5 signer onboarding

Open a step below. Each dApp has its own Testnet/Mainnet switch (testnet default; mainnet is a deliberate red-banner Tier 2 switch). Follow the numbered order the first time through.

Lock STX toward a signer via pox-5 stake — for a delegator or a signer staking their own STX. Shows currently-staked amount and registration.

Confirm registration, grant validity, delegated amount vs minimum, and recent activity. No wallet needed.
→
Guide + mainnet checklist → signer-sidekick repo ↗
Two keys, kept apart: the signer key lives on the node (step 2, more exposed); the manager-admin wallet is the funded, hardware-backed key you connect in steps 1/3/4, and it becomes the manager's admin on deploy. Never paste the signer private key into these pages — they only need the public key.

Staking has no pool manager and no aggregate-commit: anyone stakes to a signer by naming its signer-manager in stake (step 5); the lock takes effect next cycle.

Network: PoX-5 pinned per network — ST000000000000000000002AMW42H.pox-5 (testnet), SP000000000000000000002Q6VF78.pox-5 (mainnet).

Opening these: double-clicking works for navigation, but wallet connection and API reads are most reliable when the folder is served over http:// (e.g. python3 -m http.server in this folder), since some wallets restrict file:// origins.
