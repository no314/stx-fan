Signer-manager admin

Advanced and maintenance calls that do not belong to the main flow. Deploy, admin rotation, and the pool fee are in the deploy tool; registration is in register-self; reward claims are in claim rewards. Each card states which wallet is allowed to call it. The fee and withdrawal cards use the manager-admin wallet. Revoke, at the bottom, is the exception and needs the signer key itself.

Sweep fee-refund dust

Called by: a manager admin. Sweeps orphaned sBTC dust that cannot be attributed to a staker.

Only sweeps the balance above what is reserved for stakers (accrued fees, unclaimed rewards, and live withdrawal liability), so it can never take staker funds.i

Withdrawal maintenance

Called by: anyone (permissionless). Settles an L1 sBTC withdrawal request by its id.

Reclaim returns a rejected withdrawal's full amount to the staker who earned it. Settle retires an accepted withdrawal so its leftover fee dust becomes sweepable. Each applies to the opposite request state.i

txid

Revoke a signer-key grant

This one needs the signer key. Unlike every other tool in this set, revoke can only be signed by the signer key itself: pox-5 requires the caller to be the wallet whose address is derived from the signer key. That means importing and connecting the signer private key, the key that normally stays on your node and never touches a website. For now use this only on testnet, or with a throwaway key. A later version is planned to sign this offline on the node and only broadcast the signed transaction here, so the key never enters the browser.

Cancel a signer key's permission to register under a given manager. Use it when the signer is a separate party from the pool operator and wants to stop that operator from registering (or re-registering) the key, or to retire a leftover authorization so it cannot be used again.i

Called by: the signer key holder (the wallet whose address is derived from the signer key). Not the manager admin.
