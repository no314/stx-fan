Deploy signer-manager

Fresh setup. Deploys the reviewed signer-manager Clarity contract. The deployer's address becomes the manager admin. Source and name must come from your reviewed render — this dApp deploys exactly what you paste and shows its hash.

Deploy with a software (hot) wallet — not Ledger (for now). This contract uses Clarity-4+ constructs (e.g. as-contract?), so it must be deployed as a Clarity-6 VersionedSmartContract (payload 0x06, clarityVersion: 6). The Ledger Stacks app cannot sign the 0x06 payload yet and returns "ledger has rejected the payload". Deploying with the version omitted (0x01) does not work either — the node then analyses under this network's default Clarity (v3), where as-contract? is unresolved and the deploy aborts at analysis (fee burned, nothing published).

Path that works: deploy from a hot wallet, then use Admin rotation below to move admin to your Ledger/cold key. update-admin is an ordinary contract-call (0x02) that the Ledger signs fine. This is temporary — once the Ledger Stacks app supports Clarity 6, the signer-manager can be deployed directly from hardware.

Admin rotation (update-admin)

Move admin control from the hot deploy key to a cold key (Ledger/multisig). This is a normal contract-call and the Ledger can sign it.

  1. 1. From the hot deploy key: add the cold key — admin <cold>, enabled true. Two admins now.
  2. 2. Reconnect as the cold key and remove the hot key — admin <hot-deployer>, enabled false. This proves the cold key works and removes the hot key in one call.
  3. 3. Treat the hot deploy key as retired.
Do not disable the last remaining admin before a replacement admin has successfully called update-admin at least once. update-admin requires an existing admin — bricking the only admin permanently freezes fees/registration (no funds lost, but unrecoverable admin surface).
